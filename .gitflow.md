# GitFlow Workflow cho AnaOi Project

## Cấu trúc Branches

### Main Branches
- **main**: Branch production, chỉ chứa code đã được test và sẵn sàng deploy
- **develop**: Branch development, chứa code đang phát triển

### Supporting Branches
- **feature/**: Từ develop, dùng cho tính năng mới
- **release/**: Từ develop, chuẩn bị release
- **hotfix/**: Từ main, sửa lỗi khẩn cấp

## Quy trình làm việc

### 1. Tạo Feature Branch
```bash
# Từ develop
git checkout develop
git pull origin develop
git checkout -b feature/tên-tính-năng
```

### 2. Phát triển và commit
```bash
git add .
git commit -m "feat: mô tả tính năng"
git push origin feature/tên-tính-năng
```

### 3. Merge vào develop
- Tạo Pull Request từ `feature/` → `develop`
- Review code
- Merge sau khi approved

### 4. Release
```bash
# Từ develop
git checkout develop
git pull origin develop
git checkout -b release/v1.0.0
# Fix bugs, update version
git checkout develop
git merge release/v1.0.0
git checkout main
git merge release/v1.0.0
git tag v1.0.0
git push origin main --tags
```

### 5. Hotfix
```bash
# Từ main
git checkout main
git checkout -b hotfix/tên-lỗi
# Fix bug
git checkout main
git merge hotfix/tên-lỗi
git checkout develop
git merge hotfix/tên-lỗi
```

## CI/CD

### Automatic Deployment
- **Push to `main`**: Tự động deploy lên production
- **Push to `develop`**: Tự động deploy lên staging (nếu có)

### Manual Deployment
- Sử dụng GitHub Actions "workflow_dispatch" để deploy thủ công

## Commit Message Convention

- `feat:` Tính năng mới
- `fix:` Sửa lỗi
- `docs:` Cập nhật tài liệu
- `style:` Format code, không ảnh hưởng logic
- `refactor:` Refactor code
- `test:` Thêm test
- `chore:` Cập nhật build, dependencies

## Branch Protection Rules

Nên set up branch protection cho:
- `main`: Require PR, require review, require status checks
- `develop`: Require PR, require status checks
